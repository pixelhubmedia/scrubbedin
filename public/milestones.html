  • Create waitlist_signups table with RLS policies
  • Wire waitlist form to Supabase insert

! Risks / Decisions

  • Patient data safety must be enforced at every layer — feed, posts, search
  • No NHS branding; platform is independent — confirm legal position early
  • Hospital directory requires admin approval to prevent spam/fake entries
  • Endorsements limited to connections — prevents gaming the system
  • No file attachments in messages — reduces data exposure risk
  • Supabase RLS policies needed before any user-facing auth goes live
  • Privacy settings must be granular from day one — retrofitting is hard
